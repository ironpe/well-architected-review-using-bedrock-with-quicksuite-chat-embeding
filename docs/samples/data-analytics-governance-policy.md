# 데이터 분석 거버넌스 정책

**문서 번호:** GOV-DA-2026-001  
**버전:** 1.0  
**시행일:** 2026-03-01  
**작성자:** 정보보안팀 / 데이터관리팀  
**승인자:** CTO / CISO  
**분류:** 내부 기밀

---

## 1. 목적 및 적용 범위

### 1.1 목적
본 정책은 조직 내 데이터 분석 활동에 대한 거버넌스 프레임워크를 수립하여, 데이터의 무결성·보안·규정 준수를 보장하고 데이터 기반 의사결정의 신뢰성을 확보하는 것을 목적으로 한다.

### 1.2 적용 범위
- 모든 클라우드 기반 데이터 분석 플랫폼 (AWS, Azure, GCP)
- 데이터 레이크, 데이터 웨어하우스, BI 도구
- 실시간 스트리밍 분석 및 배치 처리 파이프라인
- ML/AI 모델 학습 및 추론 환경

---

## 2. 데이터 분류 및 접근 제어

### 2.1 데이터 분류 체계

| 등급 | 분류 | 설명 | 예시 |
|------|------|------|------|
| L1 | 공개 | 외부 공개 가능 데이터 | 제품 카탈로그, 공시 정보 |
| L2 | 내부 | 사내 업무용 데이터 | 매출 통계, 운영 지표 |
| L3 | 기밀 | 접근 제한 필요 데이터 | 고객 행동 분석, 재무 데이터 |
| L4 | 극비 | 최소 인원만 접근 가능 | 개인식별정보(PII), 의료 데이터 |

### 2.2 접근 제어 원칙
- 최소 권한 원칙(Least Privilege) 적용
- 역할 기반 접근 제어(RBAC) 필수 구현
- L3 이상 데이터는 MFA 인증 필수
- 데이터 접근 시 목적 명시 및 승인 절차 준수

### 2.3 접근 권한 매트릭스

| 역할 | L1 | L2 | L3 | L4 |
|------|----|----|----|----|
| 데이터 엔지니어 | 읽기/쓰기 | 읽기/쓰기 | 읽기(승인 후) | 불가 |
| 데이터 분석가 | 읽기 | 읽기 | 읽기(승인 후) | 불가 |
| 데이터 사이언티스트 | 읽기 | 읽기 | 읽기(승인 후) | 비식별화 후 읽기 |
| 비즈니스 사용자 | 읽기 | 대시보드만 | 불가 | 불가 |
| 데이터 관리자 | 전체 | 전체 | 전체 | 읽기(승인 후) |

## 3. 데이터 파이프라인 보안

### 3.1 전송 중 보안
- 모든 데이터 전송은 TLS 1.2 이상 암호화 필수
- VPC 내부 통신에도 암호화 적용 권장
- 외부 데이터 소스 연동 시 VPN 또는 Private Link 사용

### 3.2 저장 시 보안
- 저장 데이터는 AES-256 암호화 필수
- 암호화 키는 AWS KMS 또는 동등 수준의 키 관리 서비스 사용
- 키 로테이션 주기: 90일

### 3.3 파이프라인 무결성
- 데이터 파이프라인 변경 시 코드 리뷰 필수
- CI/CD 파이프라인을 통한 배포만 허용
- 프로덕션 환경 직접 수정 금지

## 4. 개인정보 보호 및 규정 준수

### 4.1 개인정보 처리 원칙
- 개인정보보호법, GDPR 등 관련 법규 준수
- 분석 목적의 개인정보 사용 시 비식별화 처리 필수
- 비식별화 기법: k-익명성(k≥5), 차분 프라이버시 적용

### 4.2 데이터 보존 및 파기
- 분석 완료 후 원본 데이터 보존 기간: 최대 1년
- 임시 분석 데이터: 프로젝트 종료 후 30일 이내 파기
- 파기 시 복구 불가능한 방법 사용 (암호화 키 폐기 포함)

### 4.3 동의 및 고지
- 고객 데이터 분석 시 개인정보 처리 동의 확인
- 프로파일링 목적 분석 시 별도 동의 필요
- 분석 결과의 자동화된 의사결정 적용 시 고지 의무

## 5. 클라우드 아키텍처 거버넌스

### 5.1 AWS Well-Architected 프레임워크 준수

#### 운영 우수성 (Operational Excellence)
- 데이터 파이프라인 모니터링 대시보드 구축 필수
- 장애 대응 런북(Runbook) 작성 및 정기 훈련
- 변경 관리 프로세스 준수

#### 보안 (Security)
- IAM 정책 정기 감사 (분기별)
- 데이터 접근 로그 최소 2년 보관
- 보안 사고 발생 시 24시간 이내 보고

#### 안정성 (Reliability)
- 데이터 파이프라인 SLA: 99.9% 가용성
- 재해 복구(DR) 계획 수립 및 연 2회 테스트
- RPO: 1시간, RTO: 4시간

#### 성능 효율성 (Performance Efficiency)
- 쿼리 성능 기준: 대시보드 로딩 5초 이내
- 배치 처리 SLA: 일일 배치 06:00 KST 완료
- 리소스 사용률 월간 리뷰

#### 비용 최적화 (Cost Optimization)
- 월간 클라우드 비용 리뷰 필수
- 미사용 리소스 자동 정리 (30일 미사용 시)
- 예약 인스턴스 / Savings Plan 활용률 70% 이상 유지

#### 지속 가능성 (Sustainability)
- 에너지 효율적인 리전 우선 선택
- 불필요한 데이터 복제 최소화
- 콜드 데이터 자동 아카이빙 정책 적용

### 5.2 인프라 코드(IaC) 관리
- 모든 인프라는 CloudFormation 또는 Terraform으로 관리
- 수동 콘솔 변경 금지 (긴급 상황 제외, 사후 IaC 반영 필수)
- IaC 코드 리뷰 시 보안 스캔 도구 적용

## 6. 데이터 품질 관리

### 6.1 데이터 품질 지표

| 지표 | 기준 | 측정 주기 |
|------|------|-----------|
| 완전성(Completeness) | 필수 필드 누락률 < 1% | 일간 |
| 정확성(Accuracy) | 오류율 < 0.5% | 주간 |
| 일관성(Consistency) | 소스 간 불일치율 < 0.1% | 일간 |
| 적시성(Timeliness) | SLA 내 데이터 도착률 > 99% | 일간 |
| 유일성(Uniqueness) | 중복률 < 0.01% | 주간 |

### 6.2 데이터 품질 프로세스
- 데이터 수집 단계에서 스키마 검증 필수
- 이상치 탐지 자동화 파이프라인 구축
- 품질 이슈 발생 시 데이터 스튜어드에게 자동 알림
- 월간 데이터 품질 리포트 발행

---

## 7. 감사 및 모니터링

### 7.1 로깅 요구사항
- 모든 데이터 접근 이벤트 로깅 (CloudTrail, VPC Flow Logs)
- 쿼리 실행 로그 보관 (최소 1년)
- 관리자 활동 로그 별도 보관 (최소 3년)

### 7.2 모니터링 및 알림
- 비정상 데이터 접근 패턴 실시간 탐지
- 대량 데이터 다운로드 시 자동 알림 (임계값: 10GB)
- 권한 변경 시 보안팀 자동 통보

### 7.3 정기 감사
- 내부 감사: 반기 1회
- 외부 감사: 연 1회
- 접근 권한 리뷰: 분기 1회
